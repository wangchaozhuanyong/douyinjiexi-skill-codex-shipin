# HyperFrames Components

Use these reusable component patterns before inventing new scene layouts.

1. `ColdOpenProofCard`
   Pain headline plus one proof image in the first 3 seconds.

2. `ProofWallGrid`
   A 2x2 or 3-card grid of real outputs, screenshots, or command results.

3. `UIStepHighlight`
   Real UI screenshot or recording with cursor, highlight box, and one short label.

4. `BeforeAfterCompare`
   Left side shows the weak method, right side shows the corrected result.

5. `MistakeCorrectionCard`
   "Do not do this" and "change it to this" in a compact card.

6. `PromptTemplateCard`
   A copyable prompt template with labeled slots.

7. `ChecklistBuild`
   A checklist that appears one item at a time as the narration explains it.

8. `TimelineProcess`
   A 3-step horizontal or vertical process with one proof frame per step.

9. `ResultReveal`
   Hide the output first, then reveal it after the method is shown.

10. `FinalRecapCTA`
   End with a recap and a save reason. Do not pressure likes or comments.

## Rules

- Match the component to the shot type.
- Do not use the same bottom caption card for the whole video.
- Keep UI screenshots readable at phone size.
- Add light sound effects for page changes, proof wall assembly, cursor clicks, and result reveals. Keep them below narration volume.
- Avoid ordinary Ken Burns zoom as the main motion.
